# ESP32\-C3 热点配网 \+ WiFi联网 \+ 外网IP获取 \+ MQTT上报 完整方案

**项目版本**：V1\.0

**适配芯片**：ESP32\-C3 系列

**开发框架**：Arduino ESP32 / ESP\-IDF（兼容双框架，本文基于Arduino演示）

**实现功能**：设备热点配网 → 自动连接用户WiFi → 解析获取设备外网公网IP → MQTT服务器定时上报IP及设备状态

**核心优势**：无屏幕、无按键、零代码修改配网，手机扫码/手动连接热点即可完成WiFi配置，上电自动联网上报，适配物联网量产设备场景

## 一、项目整体架构

### 1\.1 功能流程总图

设备上电启动 → 检测是否已保存WiFi配置 → 无配置则开启AP热点配网模式 → 手机连接设备热点、输入家庭WiFi账号密码 → 设备保存配置并重启/自动重连 → 连接本地WiFi路由 → 网络连通后请求外网IP接口 → 初始化MQTT客户端 → 定时上报外网IP、设备IP、在线状态至MQTT服务器

### 1\.2 模块拆分

- **热点配网模块**：内置Web配网页面，AP模式热点服务，WiFi信息持久化存储

- **WiFi连接模块**：自动重连、断网重试、联网状态检测

- **外网IP获取模块**：HTTP请求公共IP接口，获取公网出口IP（区别于局域网IP）

- **MQTT上报模块**：MQTT客户端初始化、心跳保活、定时数据上报、断线重连

## 三、核心功能原理说明

### 3\.1 热点配网原理

ESP32\-C3 支持AP\+STA双模切换，设备首次上电无WiFi配置时，自动开启**AP热点模式**，自建WiFi热点，手机连接热点后，访问内置Web配网页面（默认IP：192\.168\.4\.1），输入本地WiFi的SSID和密码，提交后设备将信息存入EEPROM，自动切换为STA模式，连接用户路由器WiFi。

配置信息永久保存，后续上电自动连接，无需重复配网；如需更换WiFi，可长按自定义按键或清空EEPROM数据重置配网。

### 3\.2 外网IP获取原理

ESP32局域网IP仅为内网地址，无法反映设备公网出口IP。设备联网成功后，通过HTTP请求公共免费IP查询接口（如ip\.sb、icanhazip\.com），解析返回的JSON/文本数据，获取设备真实外网公网IP，用于云端设备溯源、网络环境识别。

方案采用多接口容错机制，单个接口请求失败时自动切换备用接口，保证IP获取成功率。

### 3\.3 MQTT上报原理

设备获取外网IP后，初始化MQTT客户端，连接自定义MQTT服务器（本地私有服务器/阿里云/腾讯云公共MQTT服务），订阅上报主题，定时推送设备信息，包含：设备ID、局域网IP、外网公网IP、WiFi信号强度、设备在线状态、上报时间。同时开启MQTT断线自动重连、WiFi断网重连机制，保证设备长期稳定在线。

## 五、烧录与使用教程

### 5\.1 程序烧录

1. 打开项目源码，根据实际需求修改头部MQTT服务器地址、上报周期、热点名称等参数

2. 开发板选择 `ESP32C3 Dev Module`，选择对应端口

3. 点击上传，完成程序烧录，烧录成功后设备自动重启

### 5\.2 首次配网流程

1. 设备上电，无WiFi配置时自动开启热点 **ESP32\-C3\-NET\-CFG**

2. 手机打开WiFi，连接该热点（无需联网，仅本地连接）

3. 手机自动弹出配网页面，或浏览器访问 `192.168.4.1`

4. 在页面中输入**2\.4G WiFi名称、WiFi密码**，点击提交配置

5. 设备自动保存配置，关闭热点，切换至STA模式连接路由器WiFi

6. 串口打印联网信息、局域网IP、外网IP，初始化MQTT连接

## 六、重置配网方法

如需更换WiFi路由器或重置配置，两种方式：

1. **软件重置**：串口发送重置指令，或代码中开启按键触发EEPROM清空，重启设备重新进入配网模式

2. **烧录重置**：重新烧录程序，勾选清空Flash数据，清除旧WiFi配置

## 七、常见问题排查

### 7\.1 配网成功后无法联网

- 检查WiFi是否为**5G频段**，ESP32\-C3仅支持2\.4G WiFi

- 核对配网输入的WiFi账号密码是否正确，区分大小写

- 检查路由器是否开启MAC地址过滤、设备接入限制

### 7\.2 外网IP获取失败

- 确认设备已正常联网，可串口打印排查WiFi连接状态

- 检查网络防火墙是否拦截HTTP外网请求

- 替换源码中的公共IP接口，切换备用域名

### 7\.3 MQTT连接失败、无数据上报

- 核对MQTT服务器地址、端口是否正确，公网服务器需开启端口放行

- 检查MQTT服务器是否开启匿名访问，或补充账号密码认证逻辑

- 串口打印MQTT连接状态码，排查断线原因

- 排查设备网络是否可以正常访问外网/内网MQTT服务器

### 7\.4 设备反复重启、配网失效

- 检查供电电压是否稳定，USB供电不足会导致设备重启

- EEPROM写入异常，清空Flash重新烧录程序
